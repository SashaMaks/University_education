## Задание 2

#### Желтые страницы.
Необходимо разработать сущность ТелефонныйСправочник, который будет использоваться для
создания справочника со списком телефонов и имен, с целью поиска телефона по имени. Будем
считать достаточным хранение пар телефон-строка, например “89003337788 - Вася”.
Состояние сущности описывается набором пар “телефон – имя”, причем и телефон и имя являются
строками. В целях упрощения задачи будем считать, что одному имени всегда соответствует один
телефон, и один телефон соответствует одному имени.
Инициализация сущности выполняется без аргументов.
Поведение сущности описывают следующие действия:
- Добавление новой пары “телефон – имя” (как два отдельных аргумента). Конкретное местоположение добавленной пары не важно. Если добавляемое имя уже есть в списке, то имеющийся телефон заменяется на новый, и затем старый телефон возвращается из метода.
- Удаление значения. Для удаления необходимо указать имя удаляемого контакта, после чего удаляется вся пара.
- Получение значений. Для получения необходимо указать имя контакта, тогда возвращается его телефон.
- Может быть приведен к строке. Строка должна представлять собой полный перечень всех хранимых пар.
- Можно проверить есть ли конкретный телефон или конкретное имя в списке.
- Можно узнать текущее количество контактов.
- Можно запросить в виде массива:
o Все пары
o Все телефоны
o Все имена
- Можно получить массив всех имен, указав часть названия (начиная с первого символа).
Продемонстрируйте работоспособность решения на примерах.


#### Решение:
Создан класс PhoneDirectory в котором имеются:
- конструкторы принимаемые ничего, имя телефон
- геттеры имени и номера
- метод для добавления новой пары "телефон - имя", если имя или номер пустой то будет ошибка, а если же подаваемый телефон уже был, то ничего не будет, а если же подадут имя которое было, то метод вернет старый телефон
- методы проверки наличия имени и телефона возвращающие boolean
- метод возвращающий количество контактов
- метод который удаляет пару "телефон - имя" по имени
- методы возвращающие все телефоны, все имена, все пары
- метод который возщвращает массив имен, которые начинаются на подаваемый методу префикс ("В" - "Вася","Варя")
- преобразование в строку, которое выводит объект в виде “телефон - имя \n телефон - имя ...”

#### Фигуры.
Необходимо продумать структуру и организацию следующих геометрических фигур:
1. Круг. Задается точкой координат центра и радиусом.

4. Треугольник. Задается тремя точками координат.

#### Решение: 
Создадим круг:
- конструкторы с точкой центра + радиусом и координатами точки центра + радиусом
- геттеры центра, радиуса и типа фигуры
- сеттеры центра через точку и координаты, радиуса
- метод высчитывания площади
- преобразование в строку, которое выводит объект в виде “Круг с центром {X;Y}, с радиусом _ , площадь _”

Создадим квадрат:
- конструкторы с точкой верхнего левого угла + стороной и координатами точки верхнего левого угла + стороной
- геттеры координаты верхней левой точки, всех точек, длины стороны и типа фигуры
- сеттеры координаты верхней левой точки через точку и координаты, длины стороны
- метод высчитывания площади
- преобразование в строку, которое выводит объект в виде “Квадрат в точке {X;Y}, со стороной _ , площадь _”

Создадим прямоугольник:
- конструкторы с точкой верхнего левого угла + стороной a + стороной b и координатами точки верхнего левого угла + стороной a + стороной b
- геттеры координаты верхней левой точки, длин сторон и типа фигуры, 
- сеттеры координаты верхней левой точки через точку и координаты, длины сторон
- метод высчитывания площади
- преобразование в строку, которое выводит объект в виде “Прямоугольник в точке {X;Y}, со сторонами _ _, площадь _”

Создадим треугольник:

- геттеры точек и типа фигуры
- сеттеры точек
- метод высчитывания площади

- преобразование в строку, которое выводит объект в виде “Треугольник с точками A=_, B=_, C=_, площадь _”
